I have been made aware of a very interesting data visualization : Considering places and some connection between them (i.e. roads, shortest distance as the crow flies) as stars and star constellations.

Immediately I became inspired to work at this idea with some bells and whistels, i.e. star colour, star brightness, how to draw star constellations...

The main idea of this code is as follows: Construct maps of countries with the biggest X cities, draw them as stars in the night sky, adjust brightness and size according to the population and draw star constellations with them via Minimum Spanning Trees using the distance between the cities.

I get the coordinates and population count by the geonames-API. I also need the borders of the country - imagine a country where every big city is in the center. Then the stars will only cover the center, but the night sky, too. This will give the wrong impression about the location of the cities. For this I get "smalles rectangles containing the country" from opencagedata.

Cities¶

Now that I have the country informations as needed, I can query the geonames-API for the cities. Note that I can only filter by country-code, not by country-name (the search parameter is still called country, though. I of course can also filter after getting the results, but there is the possibility that the results are so bad that the 1000 rows are wittled down to not enough rows for a meaningfull visualization. Search results can get bad if the search engine thinks that it is a relevant result from the literal query, but this is not true for the query as meant, e.g. Searching for Italy and getting cities in the USA with a district named Little Italy back. I understand why this is returned, but I don't want it.

Featureclass P means only "places of people", i.e. cities, districs... FeatureCode PLX are districts. I only want the whole city, so I filter it. See here.

Homely! The next step would be to add size and brightness according to the city population.

Size: I simply define a size for the biggest city and then scale accordingly.

Brightness: When is a star "not bright"? When it is the same colour as the night sky. So I consider the continuum of colours "between" night sky and stars. If a city is 30% as big as the biggest city, then it gets the colour 30% the way from night sky to star. So

city_colour = colour_night_sky + ratio_population_count * (colour_star - colour_night_sky)

I am not fully satisfied with the brightness approach, but it is good enough.

The dropoff in the size and brightness is too big. In many countries this would be even worse: consider France where the 2nd largest city already has only a third of the population of the biggest city, i.e. Lyon will only be a third as bright and big as Paris.

For this I introduce scaling functions, e.g. taking the n-th root of the ratio (square root: 0.81 -> 0.9, 0.36 -> 0.6).

I also represent the citys by literal stars instead of circles.

Note the sorting of the cities: I want small cities to be drawn first, so brighter cities that are roughly at the same location are drawn over them.

The stars are nice. Now onto the star constellations: A first approach would be Minimum Spanning Trees of the complete graph on the vertices. They can be fastly computed and are planar, not too sparse and not too crowded.

def get_minimum_spanning_tree(df_cities):
    df = df_cities.copy()
    df["cross_join_key"] = 1
    # < means that an unordered pair of cities is only considered once and only if the cities are not the same, i.e. 
    #I will not consider a pair (City1,City1) and I will not consider the pair (City2,City1) if I already consider (City1,City2)
    df2 = df.merge(df, on = ["cross_join_key"]) \
                .query("city_x < city_y") \
                .drop("cross_join_key", axis = 1)
            
    #euclidian distance of the coordinates
    df2["distance_squared"] = (df2.latitude_x - df2.latitude_y) ** 2  + (df2.longitude_x - df2.longitude_y) ** 2      

    #build the complete graph
    G = nx.Graph()
    for entry in df2.to_dict("records"):
        G.add_edge(entry["city_x"], entry["city_y"], weight = entry["distance_squared"])

    #build the MST. Since the input is a complete graph, every possible way of drawing will be considered
    minimum_spanning_tree = nx.minimum_spanning_tree(G)

    #transform to DataFrame with coordinates 
    df_mst = pd.DataFrame([edge for edge in minimum_spanning_tree.edges],columns = ["city_x","city_y"])
    df_mst2 = df_mst.merge(df2, on = ["city_x","city_y"])
    df_mst2 = df_mst2[["city_x","latitude_x","longitude_x","city_y","latitude_y","longitude_y"]]
    return df_mst2

Note the join on the city name. This means I have to delete duplicate city names, I will take the one with the biggest population.

I also add a filter how many citys will be displayed - the biggest count_cities cities.

I see a glaring problem, especially with Russia: The coordinates can be stretched. If we look at the axes, the x-axis covers 300 Longitude, the y-axis only 35 Latitude. The reason for this is that I try to draw the country in a (rough) square, but not every countries "containing rectangle" is close to a square.

For every country I will define an individual rectangle contained in a square. The square defines the maximum size of the plot, the rectangle the actual size.

    dict_country = df_agg_countries.query(f"country == '{country}'").to_dict("records")[0]

    df_cities["fraction"] = df_cities.population / dict_country["population"]
    df_cities["fraction_normalized"] = df_cities.fraction / df_cities.fraction.max()

    for colour in ["red","green","blue"]:
        colour_sky = reference_colours[("sky",colour)]
        colour_star = reference_colours[("star",colour)]
        df_cities[f"rgb_{colour}"] = min(colour_sky,colour_star)  \
                                          + df_cities.fraction_normalized ** (1/5) * abs(colour_sky - colour_star)
                                      


    
    #define the indivudual rectangle: First measure the country, then decide if it is a "tall" or a "wide" rectangle.
    #"Tall" rectangles are as tall as the containing square, "wide" rectangles are as wide as the containing square. The other dimension is then scaled
    country_width = dict_country["bound_northeast_longitude"] - dict_country["bound_southwest_longitude"]
    country_height = dict_country["bound_northeast_latitude"] - dict_country["bound_southwest_latitude"]
    
    scaled_width = 0
    scaled_height = 0

    if ( country_width <= country_height): #longer than tall 
        scaled_height = max_height
        scaled_width = max_width * country_width / country_height
    else: #taller then long
        scaled_width = max_width
        scaled_height = max_height * country_height / country_width      

    #If the rectangle is wide, there might be a lot of whitespace between the actual plots. For this I remember where the last _rectangle_, not square, ended
    end_of_rectangle = end_of_rectangle + scaled_height + 0.2
    rectangle = [0, - 1 * end_of_rectangle,scaled_width,scaled_height]


    ax = plt.axes(rectangle)

    #set the borders of the plot to the "borders" of the country. Not that this is not necessarily square
    ax.set_xlim(dict_country["bound_southwest_longitude"],dict_country["bound_northeast_longitude"])
    ax.set_ylim(dict_country["bound_southwest_latitude"],dict_country["bound_northeast_latitude"])
    ax.set_xlabel(f"'{country}'")
    

    #colours
    ax.set_facecolor((reference_colours[("sky","red")]
                      , reference_colours[("sky","green")]
                      , reference_colours[("sky","blue")]
                     ))
    colours = [(entry["rgb_red"],entry["rgb_green"],entry["rgb_blue"]) for entry in df_cities.to_dict("records") ]

    #sizes
    sizes = 100 * df_cities.fraction_normalized.map(scaling_function_size)


    #draw the stars
    ax.scatter(df_cities.longitude,df_cities.latitude, c = colours, s = sizes, marker = (5,1,0), zorder = 2)
    #draw the lines of the MST
    df_mst = get_minimum_spanning_Tree(df_cities)
    for edge in df_mst.to_dict("records"):
        point_1=(edge["longitude_x"],edge["latitude_x"])
        point_2=(edge["longitude_y"],edge["latitude_y"])
        ax.plot((point_1[0],point_2[0]),(point_1[1],point_2[1]), color = ((0.5,0.6,0.7)), linewidth = 0.2, zorder = 1)
